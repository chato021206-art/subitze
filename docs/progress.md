# SNOW カードコレクション — 実装進捗

## Sprint 1: カードデータ基盤＋ドロップロジック
**ステータス:** 実装完了 - 評価待ち
**実装日:** 2026-04-09

### 実装内容
- カードカタログ定義（`CARD_CATALOG`）: 全100枚分のID・難易度・ステージ番号・レアリティ・敵名のマッピング
- ドロップ判定ロジック（`rollCardDrop`）: レアリティ別確率（Common 20%, Rare 12%, Legendary 8%）、★3ボーナス×1.5倍、既取得スキップ
- カード収集状態の永続化: `snow_cards`キー（取得日付付き）、`snow_cards_new`キー（未確認フラグ）
- 結果画面への通知: カード取得時にレアリティ別デザインの通知を表示（名前・レアリティ星・収集数）
- データリセット対応: 全データ削除時にカードデータも消去

### 自己評価

| 基準 | スコア (1-5) | コメント |
|------|-------------|---------|
| 機能完全性 | 5 | 4つの受け入れ基準すべてを満たしている |
| コード品質 | 4 | 既存コードスタイルに合わせた簡潔な実装。IIFE でカタログ構築 |
| UI/UX | 4 | レアリティ別の色分け通知。Sprint 2 で演出を強化予定 |
| エラーハンドリング | 4 | localStorage 読み込みは try-catch、CARD_CATALOG 存在チェック済み |
| 既存機能との統合 | 5 | showResult に引数追加のみ。既存の結果画面表示を一切壊さない |

### 技術的な判断
- カードIDは `EASY_0`, `NORMAL+_9` のような `難易度_インデックス` 形式。シンプルでデバッグしやすい
- 収集データは `{ cardId: { date: 'YYYY-MM-DD' } }` のオブジェクト形式。取得日を保持しつつ検索が O(1)
- 未確認カード（NEW）の管理を別キー `snow_cards_new` で分離。図鑑閲覧時の消去を Sprint 4 で実装予定
- Plus版は ENEMIES の同じ配列を参照するため、`isPlus` フラグで区別
- ENEMIES[baseDiff] の存在チェックを入れ、ULTIMATE+ の独自敵定義にも対応

### 既知の課題
- 結果画面の通知はシンプルなテキスト表示。Sprint 2 でカードビジュアル付き演出に強化
- 通知が表示されない場合（ドロップなし）は何も表示しない設計。「カードドロップなし」を明示すべきか要検討

### Evaluator への引き渡し事項
- 起動方法: `stage.html` をブラウザで開く
- テスト対象URL: `stage.html`
- テストシナリオ:
  1. EASYのステージ1をクリア → 結果画面で20%の確率でカード通知が表示される
  2. カード取得後、ページリロード → ブラウザコンソールで `JSON.parse(localStorage.getItem('snow_cards'))` を実行し、取得カードが保持されていることを確認
  3. 同じステージを再クリア → 既取得カードは重複ドロップしない
  4. ステージ5（Mid-Boss）クリア → Rare確率（12%）でドロップ
  5. ステージ10（Final Boss）クリア → Legendary確率（8%）でドロップ
  6. ★3クリア → ドロップ率が1.5倍になる
  7. デバッグ: コンソールで `getCardCount()` を実行して取得枚数を確認

## Sprint 2: カードデザイン＋ドロップ演出
**ステータス:** 実装完了 - 評価待ち
**実装日:** 2026-04-09

### 実装内容
- `renderCard()` 関数: レアリティ別枠デザイン（Common=白銀、Rare=スチールブルー、Legendary=金）でカードを描画。サイズは通常(120×168)とサムネイル(60×84)に対応
- カード回転(flip)アニメーション: `rotateY`で裏→表の3D回転演出。low-endデバイスではフェードインにフォールバック
- Legendaryカード光沢エフェクト: `cardShimmer`アニメーションで斜め光が流れる演出。low-endでは静的な半透明に簡素化
- Plus版カード対応: `.is-plus`クラスでSVGに`hue-rotate(200deg) saturate(1.4) brightness(1.05)`フィルタを適用
- カード裏面デザイン: ダークテーマ背景にSNOW/CARDロゴ（図鑑の未取得表示用にも使用可能）
- 結果画面の通知をビジュアルカード付きに全面刷新

### 自己評価

| 基準 | スコア (1-5) | コメント |
|------|-------------|---------|
| 機能完全性 | 5 | 3つの受け入れ基準すべてを満たしている |
| コード品質 | 5 | renderCard は opts パターンで柔軟。図鑑(Sprint 3)でも再利用可能 |
| UI/UX | 5 | 回転アニメ+Legendary光沢で取得の喜びを演出。TCG感のあるデザイン |
| エラーハンドリング | 4 | ENEMIES 未定義時はSVG空文字にフォールバック |
| 既存機能との統合 | 5 | Sprint 1のドロップ通知領域を拡張しただけ。既存UI破損なし |

### 技術的な判断
- カードサイズ比率5:7（120×168px）はトレカ標準比率に準拠
- 3D flip は `perspective` + `rotateY` で実装。`transform-style: preserve-3d` でリアルな奥行き
- Legendary シマーは `::after` 疑似要素でオーバーレイ。DOM追加なしでパフォーマンス良好
- Plus版はSVG要素へのCSSフィルタで対応。既存のゲーム内plus-enemy表示と同じ手法で視覚的統一感を担保
- renderCard に `showBack` オプションを用意。Sprint 3 の図鑑で未取得シルエット表示に使う

### 既知の課題
- なし

### Evaluator への引き渡し事項
- 起動方法: `stage.html` をブラウザで開く
- テストシナリオ:
  1. ステージクリアでカードドロップ → 結果画面でカードが回転して登場すること
  2. Common / Rare / Legendary でフレーム色が異なること（白/銀青/金）
  3. Legendaryドロップ時に金色の光沢が流れること

  5. デバッグ: コンソールで `renderCard(CARD_CATALOG['EASY_9'], {flip:true})` を実行し document.body に追加して確認可能
