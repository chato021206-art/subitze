# SUBITZE — 完全オリジナル化＋グローバル本格展開 製品仕様書

## 概要
瞬間認識（Subitizing）を題材にした数あそび＆コレクションゲームを、**全年齢対象・買い切り型・広告なし・オフライン動作**の iOS/Android アプリとして世界展開する。現在の Web 版には既存 IP への類似や宗教モチーフ、アクセシビリティの課題があるため、**商標・宗教・文化・アクセシビリティのすべてをクリーンアップした上で、世界中の誰にでも安心して渡せる知育／脳トレアプリ**として作り直す。

**ターゲット**
- 就学前〜小学校低学年（親が購入）
- シニア層（60代以上、本人購入）
- 世界中の家庭向けギフト用途

**販売戦略**
- iOS App Store / Google Play で世界同時配信（70 ヶ国以上）
- 買い切り **480〜980円**相当、各国通貨に自動換算
- サブスクなし・広告なし・IAP なし・トラッキングなし

**品質ポリシー**
- 既存 IP への類似 = ゼロ
- 宗教・文化固有シンボル = ゼロ
- 色だけで判定を伝えない（視覚アクセシビリティ徹底）
- 5年前の中価格帯端末で快適動作

## スプリント計画

### Sprint 0: 世界観＆キャラクター再設計（完全オリジナル化）
**ゴール:** 既存 IP に一切依存しない、特定の宗教・文化に属さない、架空の世界観とキャラクター群を作り上げる。

**背景:**
- 既存の敵キャラには FF 系の類似（マンドラ、モルボル類似、チョコボ類似など）や、ヒンドゥー／仏教／神道のモチーフ（三つ目のマハカラ、鳥居、曼荼羅など）が含まれている。これらはグローバル配信時に法的リスクと文化的摩擦を生む。

**機能:**
- [ ] 5 つの難易度に対応する架空の世界観を創作（例：スフィア大陸・クリスタル海・砂のオアシス・雲上の遺跡・星の海）
  - 特定の地球文化・宗教・神話を想起させない抽象的な世界
  - 各世界にテーマカラーとムード設定
- [ ] 各難易度に 10〜11 体のオリジナルキャラクター（合計 51 体）を新規命名
  - 既存の「ファンガ／マンドラ／モルビー／チョコン」等すべて置換
  - 商標検索で登録がないか確認（日米欧）
  - 一般名詞・架空名・組み合わせ造語を基本とし、既存ゲームキャラとの類似チェック
- [ ] 各キャラのビジュアルを新デザインで作り直す
  - チョコボに似たひよこ禁止、モルボルに似た触手モンスター禁止
  - 三つ目・曼荼羅・鳥居・数珠・輪廻など宗教的装飾ゼロ
  - 武器もクリスチャン十字・ダビデの星など宗教シンボル禁止
- [ ] 各キャラに短いオリジナルのバックストーリーを付与（カード図鑑用）
  - 特定の地球文化への言及を避ける
- [ ] ボス・中ボスは既存と同じ配置構造（5 面中ボス、10 面ボス、11 面真ボス）を踏襲

**受け入れ基準:**
- 全キャラ名を米国・日本・欧州・中国の商標データベースで検索して衝突なし
- 既存ゲーム（FF／メガテン／ポケモン／ドラクエなど）のキャラと明らかに似ているものがゼロ
- 宗教・宗派を想起させるシンボル（十字架・曼荼羅・鳥居・数珠・三つ目・六芒星など）が全キャラ・背景からゼロ
- 第三者の法務チェック（外部レビュー）で OK が出る

### Sprint 1: ビジュアルアセット刷新（カード絵柄＋敵アイコン）
**ゴール:** Sprint 0 で決めたオリジナルキャラを実際のカード画像と敵 SVG に反映する。

**機能:**
- [ ] 全 102 枚のカード画像を新キャラで再生成
  - 既存画像生成パイプライン（NB2/Imagen 4）を使用
  - プロンプトから宗教・既存 IP 要素を完全排除
- [ ] 全 51 体の敵 SVG アイコンを新デザインで描き直し
- [ ] ULTIMATE / ULTIMATE+ の背景 SVG も宇宙テーマは維持しつつ、固有文化要素を排除
- [ ] カード枠デザインは既存のまま（common/uncommon/rare）

**受け入れ基準:**
- 全カード画像（102 枚）が新キャラに更新されている
- 全敵 SVG アイコン（51 体）が新デザインに差し替わっている
- 既存の hue-rotate 仕組み・NATIVE_PLUS_DIFFS 仕組みを維持したまま動作する
- カード図鑑を開いて全キャラが正しく表示される

### Sprint 2: 視覚アクセシビリティの徹底
**ゴール:** 色覚障害・弱視・色覚多様性のあるユーザーでも完全にプレイできる状態にする。

**機能:**
- [ ] 判定表示を **色＋形＋テキスト＋アイコン** の多重符号化に変更
  - EXCELLENT: 黄＋星アイコン＋「EXCELLENT」文字
  - GREAT: 青＋稲妻アイコン＋「GREAT」文字
  - GOOD: 橙＋丸アイコン＋「GOOD」文字
- [ ] タイマーバーに判定帯の境界を**縦線＋形マーカー**で追加
- [ ] カードのレア度を色枠だけでなく**形枠**（シンプル／ダブルライン／ゴールドエンブレム）でも表現
- [ ] 色覚対応モード（1型・2型・3型）のプリセット
- [ ] ハイコントラストモード（全UI）
- [ ] 最小ブロックサイズを **24px→30px** に引き上げ、視力が弱いユーザーでも見える
- [ ] 全テキストに最大 200% 文字サイズ拡大オプション

**受け入れ基準:**
- 色覚シミュレーター（1型・2型・3型）で判定の区別がつく
- ハイコントラストモード ON で全画面の要素が明確に区別できる
- 文字サイズ最大で全画面のテキストが見切れない
- WCAG 2.1 AA 相当の色コントラスト比を達成

### Sprint 3: モーション・光過敏性対応
**ゴール:** 前庭障害・光過敏性発作・ADHD など、動きや刺激に敏感なユーザーへの配慮を実装する。

**機能:**
- [ ] モーション軽減モード：アニメーションを最小化（フェードのみ、rotate/scale 禁止）
- [ ] OS の「視差効果を減らす」設定に追従
- [ ] 光過敏性発作の警告を初回起動時に表示（WCAG 2.3.1 準拠）
- [ ] 点滅・高速アニメーションの完全停止スイッチ
- [ ] ブロック登場時のアニメーションを 3 段階（標準／軽量／無し）から選択
- [ ] カードドロップ演出の簡略化オプション

**受け入れ基準:**
- モーション軽減モード ON で rotate/scale アニメーションが発生しない
- 光過敏性警告が必ず初回起動時に表示される
- OS の「モーションを減らす」設定がアプリ設定と連動する

### Sprint 4: ネイティブシェル化＋オフライン動作
**ゴール:** 現 Web 版ベースのゲームをネイティブアプリとして起動・終了・復帰が自然に動くようにする。

**機能:**
- [ ] ゲーム本体のネイティブ埋め込み
- [ ] iOS / Android 実機で起動確認
- [ ] ホームボタン→復帰でゲーム状態が維持される
- [ ] ネットワーク遮断状態でも全機能が動作
- [ ] 全画像・フォント・スクリプトをアプリに同梱
- [ ] セーブデータはローカルに永続化

**受け入れ基準:**
- iOS / Android の実機で起動してプレイできる
- 機内モードでも全機能が動作する
- アプリを強制終了→再起動で進捗が残っている

### Sprint 5: オンボーディング＋チュートリアル強化
**ゴール:** 新規ユーザーが迷わず最初の成功体験にたどり着く。

**機能:**

  - 「数を瞬間で見抜くトレーニング」
  - 「タップするだけ、難しい操作なし」

- [ ] スキップ可能、2 回目以降は表示されない
- [ ] 既存のチュートリアル（最初の問題は必ず 3 ブロック）を維持
- [ ] 最初の 3 ステージ完走まで強めのヒント
- [ ] 初回クリア時は必ずカードがドロップ

**受け入れ基準:**
- 新規インストールから初回カード獲得までの導線がスムーズ
- 10 人のテスターがゼロ知識で完走できる

### Sprint 6: 設定画面＋4言語ローカライズ

**受け入れ基準:**

## UI/UX 要件

### 画面遷移
- スプラッシュ → （初回のみオンボーディング）→ ホーム
- ホームから：ステージ選択 / カード図鑑 / 設定 / お問い合わせ（親ゲート経由）
- ステージ選択 → プレイ → 結果 → ホーム or 次ステージ

### 主要 UI コンポーネント
- 数字ブロック（形＋色の多重符号化）
- 判定バー（色＋形＋テキスト＋アイコン）
- カードドロップ演出（モーション軽減対応）
- ボスイントロ（光過敏性対応）
- 図鑑グリッド
- 設定リスト（OSネイティブ風）
- 親ゲートダイアログ

### 画面方向
- iPhone 縦固定／iPad 縦横／Android Phone 縦固定／Tablet 縦横

### ユーザーフロー
- 新規ユーザー → オンボーディング → チュートリアル3ブロック成功 → カード獲得 → 「もう一回」
- シニア → 文字サイズ特大設定 → EASY から進行
- 視覚障害ユーザー → 色覚モード選択 → 形＋テキスト判定で完走
- 機種変更 → バックアップ → 新端末で復元

## 非機能要件

### パフォーマンス
- 起動 3 秒以内
- ステージ開始 0.5 秒以内
- 60fps 維持（5 年前の中価格帯端末）

### アクセシビリティ
- WCAG 2.1 AA 準拠
- 色覚対応 1/2/3 型
- ハイコントラストモード
- 文字サイズ最大 200%
- VoiceOver / TalkBack 対応
- モーション軽減モード
- 光過敏性発作警告

### 対応 OS
- iOS 15 以降
- iPadOS 15 以降
- Android 8.0（API 26）以降

### 多言語
- 必須：日本語・英語・簡体字・繁体字
- 将来拡張：韓国語・スペイン語・ポルトガル語・ドイツ語・フランス語・インドネシア語

### プライバシー・セキュリティ
- 広告なし
- 第三者トラッキングなし
- 個人情報収集なし
- COPPA / GDPR / 改正個人情報保護法準拠
- クラウドバックアップのみネットワーク使用（任意）

### オフライン動作
- 全ゲームモードがオフラインで動作
- 画像・フォントを全同梱

---

## 制約事項（スコープ外）

### スコープ外
- マルチプレイ / オンライン対戦（買い切り×オフライン方針と両立しにくい）
- 課金アイテム（ブースター等、買い切り一本）
- SNS 共有機能（子供の SNS 露出リスク）
- ユーザー生成コンテンツ（モデレーションコスト）
- グローバルランキング（競争より学習を優先）
- ペアレンタルダッシュボード（ポストローンチで検討）

### 既知の制約
- 既存の拡張モード（▲ 重力・サバイバル・TA・脳トレ）は機能としては維持するが、リリース時の優先度はメインステージモードより下
- カード画像の生成は Python ベースのツールを継続使用
- Apple / Google の審査ポリシー変更には個別対応
