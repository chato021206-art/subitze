# SNOW プロマイドカード コレクションシステム

## 概要
ステージクリア時に一定確率で敵キャラクターの「プロマイドカード」がドロップするコレクション要素。ラグナロクオンラインのカードシステムにインスパイアされた純粋な収集システムで、ゲームプレイへの影響（装備効果等）はない。全100種類のカードを図鑑UIで管理し、コンプリートを目指す周回動機を提供する。

## コア機能一覧

| # | 機能名 | 説明 | 優先度 |
|---|--------|------|--------|
| 1 | カードドロップ | ステージクリア時に確率でカードを獲得 | Must |
| 2 | カードドロップ演出 | 結果画面でカード獲得時の特別演出 | Must |
| 3 | カード図鑑（基本） | 収集したカードの一覧表示・閲覧 | Must |
| 4 | カード図鑑（未取得表示） | 未取得カードのシルエット表示 | Must |
| 5 | カード詳細表示 | カードをタップして拡大・詳細情報を表示 | Must |
| 6 | カードデザイン | レアリティ別の枠・背景デザイン | Must |
| 7 | カード収集の永続化 | 収集状況をlocalStorageに保存 | Must |
| 8 | 図鑑への導線 | マップ画面から図鑑にアクセスするUI | Must |
| 9 | 収集アチーブメント | カード枚数に応じたマイルストーン報酬 | Should |
| 10 | 図鑑フィルタ・統計 | 難易度別フィルタ、収集率表示 | Should |
| 11 | NEW バッジ | 新規取得カードの未確認マーク | Should |
| 12 | カード獲得履歴 | 最近獲得したカードの履歴表示 | Could |

## カード設計

### 100種類の内訳
- **EASY 1〜10**（10枚）: ロジー, サニー, ミスティ, ... レガル
- **NORMAL 1〜10**（10枚）: エグゾ, キャロン, オニン, ... マルシェ
- **HARD 1〜10**（10枚）: グレイゾ, ドリップ, ... ファルマ
- **EXTREME 1〜10**（10枚）: スクルン, スプラン, ... ビルデン
- **ULTIMATE 1〜10**（10枚）: ヘルメス, アテナ, ... クロノス
- **EASY+ 1〜10**（10枚）: Plus版カラーバリアント
- **NORMAL+ 1〜10**（10枚）: Plus版カラーバリアント
- **HARD+ 1〜10**（10枚）: Plus版カラーバリアント
- **EXTREME+ 1〜10**（10枚）: Plus版カラーバリアント
- **ULTIMATE+ 1〜10**（10枚）: Plus版カラーバリアント

### レアリティ（カード枠のデザイン区分）
| レアリティ | 対象 | カード枠の特徴 |
|-----------|------|---------------|
| ★ Common | 通常ステージ（各難易度1〜4, 6〜9） | シンプルな白枠 |
| ★★ Rare | Mid-Boss（各難易度ステージ5） | 銀色の光沢枠 |
| ★★★ Legendary | Final Boss（各難易度ステージ10） | 金色の光沢枠＋微細な輝きエフェクト |

### ドロップ確率
| 条件 | ドロップ率 |
|------|----------|
| 全ステージ クリア（通常 / Mid-Boss / Final Boss 共通） | 20% |
| ★3クリア時ボーナス | 上記 × 1.5倍（=30%） |
| 既に持っているカード | 何度でもドロップする（重複あり） |

※ レアリティ差はなく、全ステージ一律20%のシンプルな設計。★3クリアで30%に上昇するため、丁寧にクリアするほどコレクションが進む。重複ドロップを許容することで、お気に入りの敵から好きなだけカードが手に入り、周回プレイに継続的な手応えを与える。

### 重複カードの扱い
- 同じ敵カードは何度でも取得できる。
- 図鑑では取得済みカードに「所持枚数」を表示する（例: ×3）。
- 図鑑のコンプリート判定（収集率 N/100）は **ユニーク種類数** で集計する（重複は1としてカウント）。
- アチーブメント判定もユニーク種類数ベース。

### カードのビジュアル
- 通常のゲーム内SVG敵キャラより大きく、ポートレート構図で描く
- カード内にキャラ名、難易度、レアリティ星を表示
- Plus版カードはベースと同じ構図だが色調が異なる（hue-rotate適用）
- カード裏面デザイン: 共通のSNOWロゴ柄（未取得シルエット時に使用）

## スプリント計画

### Sprint 1: カードデータ基盤＋ドロップロジック
**ゴール:** ステージクリア時にカードがドロップする仕組みを実装し、取得状況を永続化する
**機能:**
- [ ] カードデータ定義: 100枚分のカードID・所属難易度・ステージ番号・レアリティのマッピング定義
- [ ] ドロップ判定ロジック: ステージクリア時にレアリティ別確率でドロップ判定。★3ボーナス倍率適用。既取得カードはスキップ
- [ ] カード収集状態の保存: localStorage（`snow_cards`キー）に取得済みカードIDセットを保存・読み込み
- [ ] 結果画面へのドロップ通知: カード取得時に結果画面（result-overlay）にカード名とレアリティを表示するシンプルな通知
**受け入れ基準:**
- ステージクリア後、確率に基づいてカードが取得できること
- 取得したカードがページリロード後も保持されていること
- 既に持っているカードが重複ドロップしないこと
- 結果画面にカード取得の通知が表示されること

### Sprint 2: カードデザイン＋ドロップ演出
**ゴール:** カード取得時の演出と、レアリティ別のカードビジュアルを実装する
**機能:**
- [ ] カードコンポーネント: レアリティ別の枠デザイン（Common=白、Rare=銀、Legendary=金）でカードを描画する関数。敵SVGをカード内にポートレート構図で配置
- [ ] ドロップ演出: 結果画面でカード取得時にカードが回転しながら出現するアニメーション。Legendary時は特別な輝きエフェクト追加
- [ ] Plus版カード対応: Plus難易度のカードはベースSVGにhue-rotate色調変換を適用した状態でカードに表示
**受け入れ基準:**
- 3段階のレアリティで枠デザインが明確に異なること
- カード取得時のアニメーションがスムーズに再生されること（low-endデバイスでもカクつかない）
- Plus版カードが通常版と視覚的に区別できること

### Sprint 3: カード図鑑UI（基本）
**ゴール:** 収集したカードを一覧で閲覧できる図鑑画面を実装する
**機能:**
- [ ] 図鑑画面: 全100枚のカードをグリッド表示するモーダル画面。難易度タブ（EASY〜ULTIMATE+）で切り替え
- [ ] 取得済みカード表示: カードイラスト＋キャラ名をサムネイルで表示
- [ ] 未取得カード表示: カード裏面デザイン（シルエット）＋「???」表示。ステージ番号のみヒントとして表示
- [ ] 図鑑への導線: マップ画面のモード選択パネルに「カード図鑑」ボタンを追加（既存の5モードの横に配置）
- [ ] 収集率表示: 画面上部に「取得数 / 100」と進捗バーを表示
**受け入れ基準:**
- マップ画面から1タップで図鑑画面にアクセスできること
- 難易度タブ切り替えで対応する10枚が表示されること
- 取得済みと未取得がひと目で区別できること
- 収集率が正しく計算・表示されること

### Sprint 4: カード詳細表示＋NEWバッジ
**ゴール:** カードの詳細閲覧と新規取得の未確認マークを実装する
**機能:**
- [ ] カード詳細モーダル: 図鑑でカードをタップすると拡大表示。カードイラスト（大）、キャラ名、難易度、レアリティ、取得日を表示
- [ ] 未取得カード詳細: シルエット状態のカードをタップすると「このカードは [難易度] ステージ [N] で入手できます」のヒント表示
- [ ] NEWバッジ: 新たに取得したカードに「NEW」マークを表示。図鑑で該当カードを閲覧すると消える
- [ ] 図鑑ボタンへのNEW通知: 未確認カードがある場合、マップ画面の図鑑ボタンにドット通知を表示
**受け入れ基準:**
- 取得済みカードをタップして詳細情報が閲覧できること
- 未取得カードをタップするとドロップ元のヒントが表示されること
- 新規取得カードにNEWマークが付き、閲覧後に消えること
- 図鑑ボタンのドット通知が未確認カードの有無に連動すること

### Sprint 5: 収集アチーブメント
**ゴール:** カード収集のマイルストーンに応じたアチーブメントを実装する
**機能:**
- [ ] アチーブメント定義: 以下のマイルストーンを設定
  - 「はじめの一枚」: カードを1枚取得
  - 「コレクター見習い」: カードを10枚取得
  - 「カードマニア」: カードを30枚取得
  - 「ハーフコンプ」: カードを50枚取得
  - 「マスターコレクター」: カードを80枚取得
  - 「コンプリート!!」: 全100枚取得
  - 「[難易度]制覇」: 各難易度10枚コンプ（×10種 = 10個）
  - 「ボスハンター」: 全ボスカード（Legendary）取得
- [ ] アチーブメント達成通知: 条件達成時にトースト通知を画面上部に表示
- [ ] アチーブメント一覧画面: 図鑑内にタブまたはセクションとしてアチーブメント一覧を表示。達成/未達成をアイコンで区別
- [ ] アチーブメント永続化: localStorage（`snow_achievements`キー）に達成状況を保存
**受け入れ基準:**
- カード取得時にアチーブメント条件が自動判定されること
- 達成時にトースト通知が表示されること
- アチーブメント一覧で達成/未達成が確認できること
- 達成状況がリロード後も保持されること

### Sprint 6: 図鑑フィルタ・統計＋ポリッシュ
**ゴール:** 図鑑の利便性向上と全体の仕上げ
**機能:**
- [ ] フィルタ機能: 「全て / 取得済み / 未取得」のフィルタ切り替え
- [ ] レアリティフィルタ: 「★ / ★★ / ★★★」で絞り込み
- [ ] 統計表示: 難易度別の収集率、レアリティ別の内訳、最後にカードを取得した日時
- [ ] カード獲得履歴: 最近取得した5枚を図鑑トップに表示
- [ ] 全体ポリッシュ: 画面遷移アニメーション、タップフィードバック、low-endデバイス対応の最終調整
**受け入れ基準:**
- フィルタが正しく機能し、表示が即座に切り替わること
- 統計情報が収集状況と一致すること
- low-endデバイスで図鑑画面がスムーズに動作すること

## UI/UX 要件

### 画面遷移
```
マップ画面
  ├── モード選択パネル → 「📇 カード図鑑」ボタン → 図鑑モーダル
  │                                                    ├── 難易度タブ切替
  │                                                    ├── カードタップ → 詳細モーダル
  │                                                    └── アチーブメントタブ
  └── ステージプレイ → 結果画面
                        └── カードドロップ時: ドロップ演出 → カード表示
```

### カードドロップ演出フロー
1. 結果画面の通常表示（スター、スコア、XP）が完了
2. 「カードドロップ！」のテキストが出現
3. カードが裏面の状態で画面中央に出現し、回転して表面が表示される
4. レアリティに応じたエフェクト（Legendary: 金色の放射光）
5. カード名とレアリティが下部に表示
6. タップまたは一定時間で結果画面に戻る

### 図鑑UI
- 既存のモーダルスタイル（daily-modal等）に合わせた全画面モーダル
- 上部: 収集率バー＋「取得数/100」
- 中部: 難易度タブ（横スクロール、既存のdiffタブと同スタイル）
- 下部: カードグリッド（2列×5行、縦スクロール）
- カードサムネイル: 60×84px程度、角丸、レアリティに応じた枠色

### カードデザイン
- サイズ比率: 5:7（トレーディングカード標準比率）
- 上部70%: キャラクターイラスト（敵SVGをポートレート構図で拡大配置）
- 下部30%: キャラ名、難易度ラベル、レアリティ星
- 枠幅: 4px、角丸: 12px
- Common枠: #E0E0E0（白銀）、Rare枠: #B0C4DE（ライトスチールブルー）、Legendary枠: #FFD700（ゴールド）

## 非機能要件

- **パフォーマンス:** 図鑑画面は100枚のカードを遅延読み込みなしで一括描画できること（カードSVGは既存の敵SVG関数を流用するため軽量）
- **ストレージ:** localStorage使用量は1KB以内に収めること（カードIDのビットフラグまたはJSON配列）
- **low-end対応:** カードドロップ演出はlow-endデバイス（`_isLowEnd`フラグ）で簡素化すること（回転アニメーション省略、エフェクト軽量化）
- **既存機能への影響なし:** カードシステムの追加により既存のゲームプレイ速度・結果画面表示が遅延しないこと

## 既知の不具合・修正項目

### Bugfix-1: ULTIMATE 後半でブロックが丸く表示される

**現象:**
- ULTIMATE 難易度の後半ステージ（特に最終盤の超高密度・狭幅レイアウト）で、本来は角ばった矩形であるべき問題スロットのブロックが、視覚的に強く丸みを帯びて表示される。
- ブロック1個のサイズが小さくなるほど顕著で、最小サイズ（およそ20px前後）では正方形ではなく丸に近い形になってしまう。

**原因の方向性（参考、実装は Generator 判断）:**
- ブロックの角丸が固定px値で指定されているため、ブロックの実寸が小さいときに角丸の比率が相対的に大きくなり、矩形の輪郭が失われている。

**期待される動作:**
- ブロックは難易度・サイズに関係なく、視覚的に「角ばった矩形」として認識できる形状であること。
- ブロックサイズが小さくなっても、角丸の見た目の強さ（半径とサイズの比率）が大きく崩れないこと。
- 通常サイズ（30〜40px程度）での既存の見た目は保たれること。

**受け入れ基準:**
- ULTIMATE / ULTIMATE+ の後半ステージ（具体的にはステージ8〜10、特に真ボス cols=12 / 縦14段の最大密度ステージ）でブロックを表示したとき、ブロックが「丸」ではなく明確に「矩形」と認識できること。
- EASY〜EXTREME の通常サイズブロックでは、修正前後で見た目の変化が最小限にとどまること（角丸自体は維持される）。
- low-end モード、dense レイアウト、brain モードのブロック装飾でも同様に矩形性が保たれること。

**スコープ外:**
- ブロックの色・影・アニメーションなど、形状以外の見た目変更はこの修正に含めない。

## 制約事項

- **スコープ外:**
  - カードのトレード/交換機能（マルチプレイ非対応のため）
  - カードによるゲームプレイへの効果（装備・バフ等）— 純粋なコレクション要素に限定
  - カードの合成・進化（システムの複雑化を避けるため）
  - サーバーサイド保存（localStorageのみ、クロスデバイス同期なし）
- **既知の制限:**
  - ブラウザのlocalStorageクリアでカード収集状況がリセットされる（既存の全データと同様）
  - カードイラストは既存の敵SVGをベースに流用。「よりリアルに描いた」ポートレート版SVGは各スプリントで段階的に追加する方針とする
